# Solution Walkthrough: T1558.003 (Kerberoasting)

### Step 1: Reconnaissance
During post-exploitation, you might find artifacts left by other pentesters or careless admins. Check temporary directories for data dumps.
`Get-ChildItem -Path C:\Temp -Recurse`
You will find a folder named `BloodHound_Data` containing `kerberoast_hashes.txt`.

### Step 2: Extraction
Read the contents of the text file. 
`Get-Content C:\Temp\BloodHound_Data\kerberoast_hashes.txt`
You will see a Kerberos TGS-REP hash (Type 23) associated with a Service Principal Name (SPN).

### Step 3: Offline Cracking (Simulated)
In a real scenario, you would copy this hash to your attacking machine and run Hashcat (Mode 13100). For this simulation, the file provides the simulated cracked password and the flag directly for assessment verification.
