# Solution Walkthrough: T1552.004 (Private Keys)

### Step 1: Reconnaissance
Private SSH keys grant password-less access to other systems. You should always search for files named `id_rsa`, `id_ed25519`, or `.pem` that have overly permissive access rights.
`find / -name "id_*" -type f -readable 2>/dev/null`

### Step 2: Extraction
You will locate `/opt/backups/ssh_keys/id_rsa_admin`. Because the permissions were improperly set to 644 (readable by anyone), you can view it.
`cat /opt/backups/ssh_keys/id_rsa_admin`

### Step 3: Analysis
Copy this key to your local machine. In a real scenario, you would run `chmod 600 id_rsa_admin` locally and use it to SSH into the target user's account. For this challenge, inspect the key's base64 payload to find the embedded flag.
