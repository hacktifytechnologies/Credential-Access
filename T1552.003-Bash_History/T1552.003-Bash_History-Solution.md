# Solution Walkthrough: T1552.003 (Bash History)

### Step 1: Reconnaissance
Linux command line history is a goldmine for credentials. If a user passes a password as a command-line argument (e.g., to `mysql` or `curl`), it is saved in their history file. Search for `.bash_history` files across user home directories that have read permissions.
`find /home -name ".bash_history" -readable 2>/dev/null`

### Step 2: Extraction
You will find that `/home/dev_user/.bash_history` is world-readable (644 permissions). Read the file:
`cat /home/dev_user/.bash_history`

### Step 3: Analysis
Observe the commands the user previously ran. You will see a `mysql` command where the administrator passed the root database password directly in the terminal, alongside an echo command containing the flag.
