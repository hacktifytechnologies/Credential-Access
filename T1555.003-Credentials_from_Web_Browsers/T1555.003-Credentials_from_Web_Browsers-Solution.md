# Solution Walkthrough: T1555.003 (Credentials from Web Browsers)

### Step 1: Reconnaissance
Attackers often look for browser data in user profiles. Search the system for the common Chrome database file `Login Data`.
`Get-ChildItem -Path C:\ -Filter "Login Data" -Recurse -ErrorAction SilentlyContinue`
You will find a database file located at `C:\Users\Public\AppData\Local\Google\Chrome\User Data\Default\Login Data`.

### Step 2: Extraction and Analysis
Since this is a simulated, partially decrypted database dump left by a previous tool, you do not need DPAPI to decrypt it. You can simply read the strings from the binary/text file.
`Select-String -Path "C:\Users\Public\AppData\Local\Google\Chrome\User Data\Default\Login Data" -Pattern "pass" -Context 0,2`
Alternatively, open the file in Notepad to visually inspect the remnants of the web credentials.

### Step 3: Flag Retrieval
Identify the target URL, username, password, and the flag appended to the record.
