# Solution Walkthrough: T1552.002 (Credentials in Registry)

### Step 1: Reconnaissance
System administrators occasionally use the Windows Registry to store configuration parameters, including passwords for custom applications. You can query the registry for keywords like "password", "pwd", or "cred".
Open a PowerShell terminal and search the `HKLM\SOFTWARE` hive:
`Get-ChildItem -Path HKLM:\SOFTWARE -Recurse -ErrorAction SilentlyContinue | Get-ItemProperty -Name "*password*" -ErrorAction SilentlyContinue`
*(Note: A targeted search might take time. Alternatively, browsing HKLM:\SOFTWARE manually for custom corporate folders is effective).*

### Step 2: Extraction
You will discover a custom registry key named `CorpAppSetup`. Query its contents directly:
`Get-ItemProperty -Path "HKLM:\SOFTWARE\CorpAppSetup"`

### Step 3: Analysis
The output will reveal the `ServiceAccount`, the plaintext `ServicePassword`, and the `Flag` stored as string values.
